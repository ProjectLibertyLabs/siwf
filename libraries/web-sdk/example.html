SIWF Web SDK Button Demo

Basic Usage

Here are the three button styles: primary, light, and dark.

Auto-initialization with Data Attributes

Simply add the data-siwf-button set it to your encoded signed request:

<!-- The button will auto-initialize and render --> <div data-siwf-button="<place encoded signed request here>"></div>

Environment: Testnet, Mode: Light

<!-- The button will auto-initialize and render --> <div data-siwf-button="<place encoded signed request here>" data-siwf-endpoint="testnet" data-siwf-mode="light" ></div>

Environment: Testnet, Mode: Dark

<!-- The button will auto-initialize and render --> <div data-siwf-button="<place encoded signed request here>" data-siwf-mode="dark" data-siwf-endpoint="https://custom.url/such-as-mock-for-testing" data-siwf-additional-callback-url-params="foo=111&bar=222" ></div>

Manual-initialization via JavaScript

<!-- Load the SIWF Button library --> <script src="siwf-web-sdk.js"></script> <!-- Use SiwfButton.render --> <script> SiwfButton.render( '#button-primary-testnet', encodedSignedRequest, { endpoint: 'testnet' } ); SiwfButton.render( '#button-light-testnet', encodedSignedRequest, {endpoint: 'testnet', mode: 'light' } ); SiwfButton.render( '#button-dark-testnet', encodedSignedRequest, { endpoint: 'testnet', mode: 'dark' } ); </script>
